TYDZIEŃ II

Warstwy strony internetowej

  • Warstwa 0 – treść,
  • Warstwa 1 – HTML warstwa semantyczna, nadaje sens treści strony. Tutaj wprowadzamy znaczniki,
  • Warstwa 2 – CSS warstwa prezentacji, nadaje wygląd poprzednim warstwą,
  • Warstwa 3 – JavaScript warstwa interakcji, najtrudniejszy element frontendu, wpływa na wszystkie poprzednie warstwy,

CONTENT IS THE KING –pamiętaj, że na stronach internetowych to treść jest najważniesza

HTML - HyperText Markup Language - hipertekstowy język znaczników.
Odpowiada za właściwe opakowanie treści strony. Treść strony czyli: teksty, nagłówki, linki, listy, definicje, obrazki, video, formularze, wszystko co jest nośnikiem informacji. Opakowujemy je w znaczniki,tagi i elementy. HTML nie jest językiem programowania, jest językiem znaczników.

Budowa znacznika:

<znacznik atrybut=”wartość”>treść</znaczik>

CSS – Cascading Style Sheets – kaskadowe arkusze stylów.
Odpowiadają z prezentację strony, wpływają na wygląd elementów HTML na ekranie lub na wydrukach. Składnia CSS to reguły {}

Budowa reguły CSS:

selektor {
   właściwość: wartość;
}

Podłączenie pliku .css do HTML – umieszczamy w sekcji <head>, poniżej widzimy wzór takiego podłączenia:

<link href=”style.css” rel=”stylesheet’ type=”tekst/css”>

Jak działa internet i przeglądarka

Protokół który obsługuje strony internetowe i ich zawartość to HTTP (obecnie coraz częściej HTTP/2). Komunikacja przeglądarka - serwer odbywa się na zasadzie request - response.

Odpowiedzi serwera mają różne kody:

  • -200-- wszystko OK
  • -30X-- szukasz w złym miejscu - przekieruję Cię w odpowiednie miejsce
  • -404-- błąd - nie mam tego czego potrzebujesz
  • -50X-- błąd wewnętrzny serwera (frontend nic z tym nie zrobi)

Root (/) jest istotną częścią ścieżek do zasobów - trzeba się nauczyć jak on działa.

JavaScript

JavaScript to frontendowe programowanie dzięki, któremu można wykonać wiele ciekawych i szalonych rzeczy. Implemantuje rzeczy ze specyfikacji ECMAScript. ECMAScript to ustandaryzowany skryptowy język programowania.

Przykład podłączenia pliku JS do HTML-a.

Niektóre programy obsługujemy jedynie z poziomu terminala. Znajomość obsługi terminala jest kluczowa w pracy programisty (frontendowaca czy backendowaca).

TYDZIEŃ IV

GIT.

GIT jest to system kontroli wersji. GIT składa się z repozytoriów czyli z naszego projektu, a nasz projekt to wszystkie nasze pliki stworzone lub wykorzystane do danego projektu. I tak jak sama nazwa wskazuje możemy go użyć m.in. do takich rzeczy jak: powrót do poprzedniej wersji swojego pliku, odzyskanie skazowanych danych, do zrobienia zmian i powrotu do stanu przed ich dokonaniem, dzielić się projektem w grupie.
Commit = stan projektu i opis tego co się zmieniło od ostatniego zapisu.
Commitujemy jak dokładamy jakąś widoczną zmianę do projektu. Lepiej commitować częściej niż za mało.
Kolejność działania (terminal) - git add . / git commit -m "opis zmiany" / git push

GitHub.

Github != GIT
GitHub jest jednym z serwerów który pozwala aby GIT działał zdalnie. Jest on programistyczną wizytówką więc warto aby nasze konto na GitHubie było zadbane ładne i "prawdziwe".

Pozycjonowanie elementów.

Świat stron internetowych jest płaski (prawie cały), poruszamy się w przestrzeni dwu-wymiarowej. W stylowaniu pomaga nam znacznik <div>. Nie ma on znaczenia semantycznego, jest używany tylko do stylowania i używamy go tylko gdy nie istnieje odpowiedni semantyczny znacznik. Trzeba uważać z używaniem tego znacznika gdyż często jest on nadużywany.
Ważną własnąścią w tym temacie jest własność display. Display może przyjmować m.in.: wartości block, inline i inline-block.
Block - elementy domyślnie układają się jeden pod drugim i zajmują całą szerokość.
Inline - elementy domyślnie ustawiają się koło siebie i zajmują tyle szerokości ile same mają.
Inline-block - przyjmują właściwości powyższych i tak ustawiają się obok siebie, ale możemy dowolnie zmieniać ich wysokości i szerokości.

Box model - fundamet tworzenia layoutu, czyli wyglądu strony internetowej. Wszystkie elementy na stronie internetowej są otoczone "niewidzialnym prostokątem". Na Box model składają się takie własności jak: padding, border, margin oraz oczywiście sam element.

Sposób zapisywania wymiarów dla padding, i margin:
zapis pełny:

padding-top: 4px; padding-right: 5px; padding-bottom: 6px; padding-left: 7px;

Shorthand (zapis skrócony):

margin: 4px 5px 6px 7px; margin: top right bottom left;

Gdy góra=dół i prawa strona=lewa strona:

margin: 16px 24px; margin: (góra i dół) (prawa i lewa strona);

Gdy wszystkie storny są takie same:

margin: 16px;

Dla właściwości border sposób zapisu jest podobny tylko musimy jeszcze dołożyć warotości color i style. Możemy używać skróconego jak i pełnego zapisu.

border: 2px solid yellow;

Position - pozycjonowanie elementów w przestrzeni, nie tylko za pomocą narzuconych przez przeglądarkę domyślnych układów. Pozwala na dokładne ustawienie elementu na stronie. Domyślna pozycja to static, mogą być również: relative, absolute, fixed i sticky. Dla dokładnego ustawienia dopisuje się jeszcze w CSS wartości: top, left, right i bottom.

JS: typy danych i immutability.

zmienne != wartość
Zmienna to takie połączenie ze wszechświatem wartości. Zmienna to nie wartość, ona tylko wskazuje na wartość która już istnieje. Są trzy sposoby zapisywania zmiennych w JS: LET vs CONST vs VAR. const - ma stałe połączenie z daną wartością. let - wartość w let po zdefiniowaniu może być zmieniana. Definiujemy tylko raz. Zmienne w JS przyjmują każdy typ danych.

Typy proste w JS (primitives)

  • String - tekst, otaczamy ' lub " (zalecane ')
  • Number - liczba
  • Bigint - duża liczba
  • boolean - true lub false
  • null - wartość celowo niezdefiniowana
  • undefinded - brak definicji
  • *symbol - tworzy "unikalne opakowanie"

IMMUTABILITY- jest to nazwa konceptu, który mówi, że nie możemy zmieniać wartości primitive values w JS. To co zmieniamy to płączenia między nimi, a zmiennymi.

DOM.

DOM - document object model (obiektowy model dokumentu). DOM to "złożony" w przeglądarce HTML czyli to co widzimy w DevTools. JavaScript może modyfikować DOM. Funkcja querySelector zwraca node (węzeł) z HTML-a, a selektor jest zapisywany zgodnie z CSS.

document.querySelector('selector')

JavaScript: Funkcje

Funkcja na wejściu dostaje dane czyli argumenty. Dla każdego argumentu zwraca jakąś wartość oraz posiada wzór.

Przykład funkcji:

function greet(age) {
     return age*10;
}

Funkcje w JS
Deklarujemy nazwę funkcji (nie zawsze). W środku dokonujemy obliczeń/ wykonujemy zadania. Funkcja przyjmuje argumenty (ale nie musi). Funkcja zawsze zwraca coś (nawet jak nie mamy return, wtedy zwraca undefinded)

Po co tworzymy funkcje: deklarujemy kod który się powtarza, zwiększamy czytelność kodu, łatwiejsze szukanie błędów, odpowiedzialność za wykonywanie zadania w jednym miejscu.

(Fat) arrow function - skraca zapis deklaracji funkcji, zwiększa czytelność (nie dla wszystkich, co kto lubi), nie tworzy nowego kontekstu this =>, nie ma dostępu do arguments i super, nie jest zamiennikiem "tradycyjnej" funkcji (ale w większości przypadków nie zobaczy się różnicy).

Przykład:

const greet = (age) => {
     console.log(age)
     return age*10;
}

Skrócony zapis przykład:

const greet = (age) => (age*10)

JavaScript: Obiekty.

Prawie wszystkow JS jest obiektem (nawet typy proste, mimo iż nie są obiektem - to tak się zachowują). Obiekt jest strukturą danych.

Przykład obiektu:

const deathStar = {
   diameter: 120000,
fire: (target) => {
     console.log(`${target} destroyed * `)
  },
  isOperating: true,
  levels: 357,
  name: 'Death Star',
  population: 1000,
}

JavaScript: Immutability w obiektach.

Nie możemy zmieniać wartości primitive values w JS. To co zmieniamy to połączenia między nimi, a zmiennymi.

  1. Sprawdź czy plik jest zapisany.
  2. Sprawdź czy edytujesz włąściwy plik.
  3. Sprawdź czy osdświeżyłeś przeglądarkę.
  4. Sprawdź czy nie masz błędów w Devtoolsach i w edytorze.
  5. Pogadaj z kaczką - czyli pogadaj z innym programistą.
